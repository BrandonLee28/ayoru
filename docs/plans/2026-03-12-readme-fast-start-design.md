# README Fast Start Design

**Goal:** Rewrite the README so a first-time GitHub visitor can install Ayoru, verify it works, and launch their first show without reading internal project details first.

## Audience

- New users arriving from GitHub
- People evaluating whether the install is simple
- Users who want a quick CLI or TUI example immediately

## Chosen Approach

Use a fast-start README.

That means:
- keep the top of the page short
- move installation and first-run steps above repo internals
- make requirements explicit before install
- add a small troubleshooting section for common install issues
- push contributor and maintainer details lower on the page

## Proposed Structure

1. Title and one-line description
2. Why use Ayoru
3. Requirements
4. Install
5. Quick start
6. TUI controls
7. Troubleshooting
8. Local data
9. Development
10. Release and maintainer notes

## Content Principles

- Lead with what the user can do now
- Keep steps short and copy-paste friendly
- Prefer concrete commands over abstract explanation
- Reduce internal terminology in the first half of the page
- Keep development notes available, but below the user path
